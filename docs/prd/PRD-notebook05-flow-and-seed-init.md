# PRD: Notebook 05 신규 사용자 흐름 정리 + 05/06/07 Seed Code 초기화 예시 반영

**Status:** ready-for-agent

## 요약

오케스트레이터 실행 순서를 `1 → 3 → 2`로 맞출 수 있도록, 아래 두 작업을 독립 이슈로 분리하고 실행 가능한 수준으로 명세한다.

1. Notebook 05 신규 사용자 실행 흐름 정리
2. Notebook 05/06/07 Seed code 초기화 예시(publicdatasetjihys) 반영

## 문제 정의

- `notebooks/05-multimodal-indexing.ipynb`에 신규 사용자 기준으로 혼동을 주는 이력성 주석/설명이 남아 있다.
- 05/06/07 노트북의 seed code 초기화 예시가 현재 운영 기준과 완전히 정렬되어 있지 않아 재현성이 떨어질 수 있다.
- 사용자가 지정한 오케스트레이터 순서(`1 → 3 → 2`)를 보장하려면, 1번/2번을 독립 이슈로 등록하고 선행 관계를 명확히 해야 한다.

## 해결 방향

- 1번과 2번을 별도 이슈로 발행한다.
- 1번 이슈는 Notebook 05의 신규 사용자 관점 흐름 정리만 다루고, 기능 로직 변경은 제외한다.
- 2번 이슈는 Notebook 05/06/07에서 동일한 seed 초기화 예시(`publicdatasetjihys`)를 일관되게 제시한다.
- 2번 이슈에 오케스트레이터 3번 선행 조건을 명시해 전체 순서(`1 → 3 → 2`)를 운영적으로 강제한다.

## 사용자 스토리

1. 신규 랩 사용자는 Notebook 05를 열었을 때 현재 권장 흐름과 실험/비교 설명을 즉시 구분하고 싶다.
2. 랩 운영자는 Notebook 05의 오래된 이력성 안내를 정리해 교육 중 혼선을 줄이고 싶다.
3. 신규 랩 사용자는 Notebook 05/06/07 어디에서 시작해도 같은 seed 초기화 패턴을 보고 싶다.
4. 랩 운영자는 `publicdatasetjihys` 예시를 표준으로 고정해 실행 안내를 단순화하고 싶다.
5. 오케스트레이터 담당자는 작업을 1→3→2 순서로 안전하게 수행할 수 있도록 이슈 의존성이 명확하길 원한다.
6. 리뷰어는 구현 전에 수용 기준 체크리스트로 결과를 검증 가능해야 한다.
